## 📸 图片规范

### 产品图片 (Hero级别)

| 规格 | 要求 |
|------|------|
| **用途** | 首页Hero、产品页主图 |
| **尺寸** | 2000x2000px (1:1) 或 2400x1600px (3:2) |
| **格式** | PNG (透明背景) 或 WebP |
| **背景** | 纯白 `#FFFFFF` 或 透明 |
| **风格** | Apple风格：简洁、高级、产品聚焦 |
| **光线** | 柔和自然光，无硬阴影 |
| **角度** | 正面45°或侧视，展示产品核心特征 |
| **后期** | 微调对比度，锐化边缘，保持色彩真实 |

**示例场景**：
```
Popohu Mini 产品图
- 主体居中，占画面60-70%
- 纯白背景
- 柔和顶部光照
- 产品下方10%区域留空（可放置文案）
```

### 产品图片 (详情级别)

| 规格 | 要求 |
|------|------|
| **用途** | 产品页详情图、特性展示 |
| **尺寸** | 1600x1200px (4:3) |
| **格式** | WebP |
| **背景** | 渐变 `linear-gradient(180deg, #FFFFFF 0%, #F8FAFC 100%)` |
| **构图** | 产品+使用场景示意图 |
| **文字** | 可叠加简洁标注（Inter字体，14px） |

### 官网插图/图标

| 类型 | 尺寸 | 格式 | 风格 |
|------|------|------|------|
| **功能图标** | 64x64px | SVG | SF Symbols风格 |
| **插图** | 800x600px | WebP/PNG | 扁平插画，品牌色 |
| **Logo** | 200x200px | SVG | 矢量，透明背景 |

### 图片命名规范

## 🎬 视频规范

### 产品演示视频 (Hero级别)

| 规格 | 要求 |
|------|------|
| **用途** | 首页Hero视频、产品页主视频 |
| **尺寸** | 1920x1080 (16:9) 或 1080x1920 (9:16竖屏) |
| **时长** | 15-30秒 |
| **格式** | MP4 (H.264) |
| **码率** | 5-8 Mbps (1080p) |
| **帧率** | 30fps |
| **风格** | Apple产品发布会风格 |
| **背景** | 纯色渐变或极简场景 |
| **音乐** | 氛围音乐，无版权问题 |

**内容结构**：
```
0-2s:   淡入，产品轮廓出现
2-8s:   产品360°旋转或关键特性展示
8-12s:  使用场景动画
12-15s: Logo+Slogan淡入
```

**技术要求**：
- 无黑边（letterboxing）
- 色彩空间：sRGB
- 音频：AAC 128kbps (如有)
- 文件大小：< 15MB (30秒视频)

### 功能演示视频 (详情级别)

| 规格 | 要求 |
|------|------|
| **用途** | 产品功能说明 |
| **尺寸** | 1280x720 (16:9) |
| **时长** | 30-60秒 |
| **格式** | MP4 (H.264) |
| **码率** | 3-5 Mbps |
| **风格** | 屏幕录制 + 简洁动画 |

### 视频命名规范

```
assets/videos/
├── products/
│   ├── popohu-mini-hero.mp4       # 产品主视频
│   ├── popohu-mini-setup.mp4      # 设置教程
│   └── pitopa-demo.mp4
└── company/
    └── company-intro.mp4          # 公司介绍
```

## 🔍 质量检查清单

交付前请确认：

### 图片检查
- [ ] 尺寸符合规范
- [ ] 文件格式正确
- [ ] 背景干净（纯白或透明）
- [ ] 无噪点和失真
- [ ] 色彩准确（符合品牌色）
- [ ] 锐化适中

### 视频检查
- [ ] 分辨率和码率符合规范
- [ ] 时长在规定范围内
- [ ] 无黑边或变形
- [ ] 声音清晰（如有）
- [ ] 转码流畅无卡顿
- [ ] 文件大小合理

## 📝 补充说明

### 图片优化
所有图片交付后需要：
1. 转换为WebP格式（如适用）
2. 压缩优化（保持质量）
3. 生成响应式版本（2x, 1x）
4. 添加lazy-loading属性

### 视频优化
1. 使用H.264编码以确保兼容性
2. 生成预览缩略图
3. 考虑提供低码率版本用于移动端
4. 添加视频封面图（第一帧或定制封面）
